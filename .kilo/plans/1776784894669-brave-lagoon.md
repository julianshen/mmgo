# Flowchart Rendering Review: mmgo vs mmdc

## Visual Comparison Summary

I compared the three example flowcharts rendered by both mmgo and the original Node.js mmdc. The PNG outputs are remarkably close — both produce clean, professional flowcharts. Key differences:

### Layout Dimensions

| Example | mmgo (W×H) | mmdc (W×H) | Notes |
|---------|-----------|------------|-------|
| simple (LR, 4 nodes) | 494×118 | 596×137 | mmgo 17% more compact |
| shapes (TD, 7 nodes) | 233×543 | 242×615 | mmgo 12% shorter |
| subgraphs (TB, 6 nodes) | 508×267 | 437×296 | mmgo 16% wider, 10% shorter |

### Styling Differences

| Aspect | mmgo | mmdc |
|--------|------|------|
| Node fill | White `#fff` | Light purple `#ECECFF` |
| Node stroke | Dark gray `#333` | Purple `#9370DB` |
| Stroke width | 1.5px | 1px |
| Approach | Inline styles | CSS classes + stylesheet |
| Edge rendering | `<line>` + `<path>` | All `<path>` |
| Text rendering | SVG `<text>` | HTML `<foreignObject>` |
| Accessibility | None | role, aria-roledescription |
| SVG size | 2.5–3.9 KB | 15–107 KB |
| CSS animations | No | Yes (edge animations) |

### What mmgo Gets Right
- **Compact, clean SVG output** — 4–27x smaller files than mmdc
- **Correct layout topology** — nodes placed correctly on all 3 examples
- **All node shapes render properly** — rectangle, diamond, stadium, parallelogram, cylinder
- **Edge routing is correct** — straight lines, curves for longer edges, arrow markers
- **Edge labels with white background** — readable on any background
- **Deterministic rendering** — same input always produces same output

### Visual Gaps vs mmdc
1. **Default color scheme** — mmgo uses white/gray (Mermaid's "neutral" theme), while mmdc uses purple fills/strokes (Mermaid's "default" theme). mmgo should match mmdc's default colors.
2. **Node spacing** — slightly different spacing characteristics; mmgo is generally more compact but sometimes wider for fan-out patterns.
3. **Edge paths** — mmdc uses curved paths for all edges; mmgo uses straight `<line>` for direct connections, only `<path>` for multi-segment edges.

## Code Quality Issues (by priority)

### High Priority (Functional Gaps)

#### 1. Default theme colors don't match Mermaid's default theme
- **Files:** `pkg/renderer/flowchart/theme.go`
- mmgo defaults: `fill:#fff, stroke:#333` (matches Mermaid's "neutral" theme)
- Mermaid default: `fill:#ECECFF, stroke:#9370DB` (purple-ish)
- **Fix:** Update `DefaultTheme()` to use Mermaid's actual default colors. Add separate "neutral" theme.

#### 2. No subgraph parsing (only rendering exists)
- **File:** `pkg/parser/flowchart/parser.go:31`
- The AST types (`Subgraph`) and renderer (`subgraphs.go`) exist, but the parser can't produce subgraphs from `subgraph foo [...] end` syntax.
- This means the 3rd example file (`subgraphs.mmd`) is misleadingly named — it doesn't actually contain subgraph syntax.
- **Fix:** Implement subgraph parsing: detect `subgraph` keyword, track nesting depth, collect nodes/edges within scope.

#### 3. No `style`/`classDef`/`class` directive parsing
- **File:** `pkg/parser/flowchart/parser.go:32`
- The AST (`StyleDef`, `Node.Classes`, `FlowchartDiagram.Classes`) and renderer (`style.go`) fully support these, but the parser silently ignores them.
- **Fix:** Parse `style A fill:#f9f,stroke:#333`, `classDef bold font-weight:bold`, `class A,B bold` directives.

#### 4. Self-loops render as invisible zero-length lines
- **File:** `pkg/layout/layout.go:376-378`
- `A --> A` produces two identical points, rendering nothing visible.
- **Fix:** Generate a loop-back arc path when `eid.From == eid.To`.

### Medium Priority (Correctness / Robustness)

#### 5. `stripComment` pipe toggle is fragile
- **File:** `pkg/parser/flowchart/parser.go:129-131`
- `inPipe` toggles on every `|` even inside brackets (`A[foo|bar]` breaks it).
- **Fix:** Only toggle `inPipe` when `depth == 0`.

#### 6. Hardcoded `fill:white` edge label background breaks dark themes
- **File:** `pkg/renderer/flowchart/edges.go:168`
- Edge label background is always white regardless of theme.
- **Fix:** Use `th.Background` color.

#### 7. No CSS class name sanitization
- **File:** `pkg/renderer/flowchart/style.go:22`
- Class names are interpolated directly into CSS selectors without validation.
- **Fix:** Validate class names match `[A-Za-z0-9_-]+`.

#### 8. No end-to-end integration tests with mmdc comparison
- Only unit tests and golden-file tests exist. No automated comparison against mmdc output.
- **Fix:** Add rendering comparison tests that validate layout topology, node counts, and label correctness.

### Low Priority (Polish)

#### 9. Rounded rect corner radius hardcoded to 5px
- **File:** `pkg/renderer/flowchart/nodes.go:38`
- Should scale with node size: `min(h/4, 8)`.

#### 10. Subgraph label position hardcoded
- **File:** `pkg/renderer/flowchart/subgraphs.go:68-69`
- Position `(rx+10, ry+18)` doesn't account for font size.

#### 11. No `%%{init:...}%%` directive parsing in parser
- **File:** `pkg/parser/flowchart/parser.go:32`
- These directives are stripped as comments; config values are lost.
- Note: The `pkg/output/svg/svg.go` pipeline handles init directives at the output level, so this may be by design.

#### 12. Arrow head types `Open`, `Cross`, `Circle` defined but never produced
- Parser only generates `Arrow` or `None`. Renderer handles all types but they're dead code.

#### 13. Variable shadowing in `position.initialPacking`
- **File:** `pkg/layout/internal/position/position.go:133`
- Inner loop variable `i` shadows outer loop variable.

#### 14. `dummy.IsDummy` uses fragile prefix check
- **File:** `pkg/layout/internal/dummy/dummy.go:91-93`
- Should use `strings.HasPrefix`.

#### 15. `compareEdgeIDs` uses `a.ID - b.ID` (potential overflow)
- **File:** `pkg/layout/internal/layoututil/layoututil.go:27`
- Use `cmp.Compare` instead.

#### 16. No XML declaration suppression option
- **File:** `pkg/renderer/flowchart/renderer.go:95`
- Always emits `<?xml?>` declaration; should be optional for inline SVG use.

## Recommended Improvement Plan

### Phase 1: Match Mermaid Default Style (Quick Win)
1. Update `DefaultTheme()` in `theme.go` to use Mermaid's default colors (`#ECECFF` fill, `#9370DB` stroke)
2. Add `NeutralTheme()` for current white/gray styling
3. Fix edge label background to use theme background color

### Phase 2: Parser Feature Gaps
1. Implement subgraph parsing (`subgraph title [...] end`)
2. Implement `style`, `classDef`, `class` directive parsing
3. Fix `stripComment` pipe-toggle bug
4. Add `-x`/`-o` arrow endpoint parsing

### Phase 3: Layout & Rendering Polish
1. Add self-loop arc rendering
2. Add CSS class name sanitization
3. Scale rounded rect radius with node size
4. Add `%%{init:...}%%` directive parsing in parser (if needed beyond svg.go handling)

### Phase 4: Testing
1. Add integration tests comparing layout topology against mmdc output
2. Add rendering regression tests with more complex flowcharts (cycles, parallel edges, long chains)
3. Add benchmark tests for performance tracking
